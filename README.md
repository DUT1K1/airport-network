# Global Flight Network Dashboard

A graph-database dashboard built on **Neo4j 5** + **NeoDash 2.4**, using the
[OpenFlights](https://github.com/jpatokal/openflights) dataset. It analyses the worldwide
airport/route network: busiest hubs, route-distance distribution, country-to-country
connectivity, and shortest-distance flight routing.

> Course project for **Graph Databases** (KIU, Semester 8).

## What's inside

```
.
├── docker-compose.yml          # Neo4j 5.26 (+APOC) and NeoDash containers
├── data/                       # OpenFlights .dat files (downloaded; mounted into Neo4j)
├── scripts/
│   ├── download_data.sh        # fetch OpenFlights data into ./data
│   └── import.sh               # run all Cypher import scripts, then print a summary
├── import/                     # Cypher import scripts (run in numeric order)
│   ├── 01_constraints.cypher
│   ├── 02_countries.cypher
│   ├── 03_airports.cypher
│   ├── 04_airlines.cypher
│   └── 05_routes.cypher
├── dashboard/
│   └── flights_dashboard.json  # the NeoDash dashboard (Load this in NeoDash)
├── report/
│   ├── REPORT.md               # written report (source)
│   ├── REPORT.html             # print-ready report → Save as PDF for submission
│   └── REPORT.pdf              # generated by you (see step 5)
└── CLAUDE.md                   # notes for AI assistants working in this repo
```

## Quick start

Requires **Docker Desktop** running.

```bash
# 1. Download the OpenFlights data (already present, re-run to refresh)
bash scripts/download_data.sh

# 2. Start Neo4j + NeoDash
docker compose up -d

# 3. Load the graph (waits for Neo4j, runs all import scripts, prints counts)
bash scripts/import.sh
```

Expected import summary:

```
airports  airlines  countries  cities  routes
7698      6161      355        7095    66771
```

### 4. Open the dashboard

1. Go to **<http://localhost:5005>** (NeoDash).
2. Click **New Dashboard → Connect**, and enter:
   - Protocol: `neo4j` · Hostname: `localhost` · Port: `7687`
   - Database: `neo4j` · Username: `neo4j` · Password: `flights123`
3. Use the **Load** button (folder icon, top-left) → **Select from file** →
   `dashboard/flights_dashboard.json`.

Neo4j Browser is also available at <http://localhost:7474> (same credentials) if you want to
run the Cypher queries by hand.

### 5. Produce the report PDF

Open `report/REPORT.html` in **Safari** (or any browser) → **File ▸ Export as PDF…**
(or `Cmd+P` → *Save as PDF*) → save as `report/REPORT.pdf`. This is the PDF to submit
alongside the dashboard.

## Notes

- **Apple Silicon:** the NeoDash image is amd64-only, so `docker-compose.yml` pins
  `platform: linux/amd64` (runs under emulation — it's just a static frontend).
- **Distances** are not in OpenFlights; they are computed in Cypher from each airport's
  coordinates via `point.distance()` and stored on every `ROUTE` relationship as `distanceKm`.
- **Shortest-distance routing** uses `apoc.algo.dijkstra` weighted by `distanceKm`.
- Credentials (`neo4j` / `flights123`) are set in `docker-compose.yml` (`NEO4J_AUTH`).

### Reset everything

```bash
docker compose down -v   # also deletes the database volume; re-run import.sh afterwards
```
