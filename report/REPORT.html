Global Flight Network Dashboard

Graph Databases (KIU, Semester 8) · David Maisuradze · Neo4j 5 + NeoDash 2.4 + Docker

1. Data & source

The dashboard is built on the OpenFlights dataset (github.com/jpatokal/openflights, Open Database License) — a well-known, community-maintained collection of global civil-aviation data. Four raw files were used:

FileRowsContents
airports.dat7,698id, name, city, country, IATA/ICAO, latitude/longitude, altitude, timezone
airlines.dat6,162id, name, IATA/ICAO, callsign, country, active flag
routes.dat67,663airline + source airport + destination airport + stops + equipment
countries.dat261country name, ISO code
planes.dat246aircraft type name, IATA/ICAO code

Flights data was chosen because it is inherently a graph — airports are nodes, routes are edges — so it showcases graph queries (hubs, connectivity, shortest paths) far better than a relational table, while staying intuitive to present.

2. Graph model

Node types (5)

  • Airport — code, name, city, country, lat/long, altitude, spatial location
  • Airline — name, IATA/ICAO, callsign, country, active
  • Country — name, isoCode
  • City — name, country
  • Plane — aircraft type name, IATA/ICAO code

Relationship types (5)

  • (:Airport)-[:LOCATED_IN]->(:Country)
  • (:Airport)-[:IN_CITY]->(:City)
  • (:City)-[:IN_COUNTRY]->(:Country)
  • (:Airline)-[:REGISTERED_IN]->(:Country)
  • (:Airport)-[:ROUTE {airline, stops, equipment, distanceKm}]->(:Airport)

Loaded totals: 7,698 airports 6,161 airlines 355 countries 7,095 cities 220 aircraft types 66,771 routes

3. Conversion & import

The raw files are headerless, comma-separated and use \N for nulls, so Neo4j reads them directly with LOAD CSV — no manual editing. The pipeline is: download (scripts/download_data.sh) → start Neo4j + NeoDash (docker compose up -d) → run five Cypher scripts (scripts/import.sh): constraints/indexes, then countries, airports + cities, airlines, routes.

Key step: OpenFlights has no distance or price column, so the great-circle distance of every route is computed in Cypher. Each airport gets a spatial point from its coordinates, and each route stores distanceKm = round(point.distance(src, dst)/1000). This derived property powers the distance histogram, the route tables and the shortest-distance routing.

4. Dashboard contents

Four pages, 41 reports, grouped by topic.

Page 1 — Airport Network Analysis

  • KPI cards; bar busiest airports; graph 12-hub network
  • Line route-distance distribution; pie top airlines
  • Table (CSV) connectivity ranking; map + density heatmap

Page 2 — Country & Route Connectivity (interactive)

  • Source & destination country selectors; KPIs
  • Graph shortest-distance path (Dijkstra, multi-hop); country-to-country network
  • Bar + table (CSV) + map of the chosen countries

Page 3 — Aviation Records & Trivia

  • Bar longest routes & busiest city pairs; pie aircraft types
  • Tables (CSV) highest-altitude, most-isolated, shortest flights

Page 4 — Airport & Airline Explorer (interactive)

  • From/To airport + airline selectors
  • Map + table destinations; value "can I fly direct?"
  • Line reachability within N stops; graph + map airline network

Tables expose CSV download; charts, graphs and maps expose PNG export.

5. Running it

bash scripts/download_data.sh     # fetch OpenFlights data
docker compose up -d              # start Neo4j + NeoDash
bash scripts/import.sh            # load the graph

Open NeoDash at http://localhost:5005, connect to neo4j://localhost:7687 (user neo4j / password flights123), then Load dashboard/flights_dashboard.json.
